Copilot CLI Sandbox

Isolated · Persistent · One command to spin up
GitHub Copilot CLI inside Docker — for developers

🐳 Docker ⚡ PowerShell 🤖 Copilot CLI 🎯 .NET + C#

What is it?

  • 🐳
    Copilot CLI in a Docker container — fully isolated, ephemeral, reproducible
  • 🔑
    Full Copilot config shared — auth, agents, skills, settings all live in ~/.copilot-sandbox/.copilot/ and are mounted into every session automatically
  • 📁
    Named sessions — each session is a host-mounted folder, files survive container exit
  • 🧩
    Optional features — bake in .NET SDKs, C# LSP, Playwright at install time
  • 🔄
    One-command updates — copilot-sandbox -Update rebuilds with latest CLI, features remembered

Prerequisites

  • 🐳
    Docker Desktop — with WSL2 backend on Windows
  • 💙
    PowerShell 6+ — works on Windows, macOS, Linux
  • 🐙
    GitHub account — with Copilot subscription or free tier access
That's it. No Node, no npm, no dotnet SDK needed on the host —
everything runs inside the container.
0:20 – 0:50

Install

# Clone the repo
git clone https://github.com/VaclavPachta/CopilotCliSandbox
cd CopilotCliSandbox

# Build with .NET 8, .NET 10 + C# Language Server
.\install.ps1 -Add dotnet8,dotnet10,csharpls
-Add bakes optional features into the image. csharpls auto-enables .NET 10 if no SDK is specified — here we're adding both explicitly.

Docker Build

# install.ps1 runs this automatically
docker build -t copilot-sandbox \
  --build-arg INSTALL_DOTNET8=true \
  --build-arg INSTALL_DOTNET10=true \
  --build-arg INSTALL_CSHARP_LS=true \
  -   # ← Dockerfile piped via stdin, no build context uploaded
Why stdin? Avoids uploading ~/.copilot-sandbox/ (session data + credentials) as Docker build context. Feature config is saved to sandbox-config.json and replayed automatically on -Update.

Authentication

Option A — Interactive
/login

Developer settings → Personal access tokens

Launch a Session

# Reload profile after install
. $PROFILE

# Start a named session AND open in VS Code
copilot-sandbox FibonacciDemo -Code
-Code calls code <session-path> before launching the container. VS Code opens the folder live — no Docker extension, no remote SSH required. Files Copilot creates inside the container appear in VS Code instantly.

Ask Copilot: TDD Fibonacci in C#

Create a new .NET solution with a class library project
and an xUnit test project.
Then implement the Fibonacci sequence in C# using a TDD
approach: write the failing tests first, then implement
the function to make them pass.
Watch the VS Code sidebar — FibonacciDemo.sln, FibonacciLib/, FibonacciTests/ appear live as Copilot works. End with dotnet test — all green ✅

Staying Current

# Rebuild with latest Copilot CLI (features remembered)
copilot-sandbox -Update

# Add a feature at update time
copilot-sandbox -Update -Add playwright

# Remove a feature
copilot-sandbox -Update -Remove dotnet8

# Add and remove in one call
copilot-sandbox -Update -Add csharpls -Remove playwright
Feature choices are saved to .copilot/sandbox-config.json and replayed on every -Update. No need to re-run install.ps1.
